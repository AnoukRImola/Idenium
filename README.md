# IDenium

**"Sign with IDenium"** — like "Sign with Google" but with ZK privacy and passport verification on Starknet.

IDenium is a zero-knowledge identity system that lets users prove attributes about themselves (age, nationality, sanctions status) without revealing personal data. Scan your passport via NFC, generate a ZK proof, verify on-chain — no trusted third party needed.

## How It Works

```
1. SCAN           2. PROVE              3. VERIFY
┌──────────┐     ┌──────────────┐      ┌──────────────┐
│ Passport │────>│ ZK Proof     │─────>│ On-Chain     │
│ NFC Scan │     │ (ZKPassport) │      │ (Garaga)     │
└──────────┘     └──────────────┘      └──────────────┘
                       │                      │
                  No personal            Nullifier only
                  data leaves            stored on-chain
                  your device
```

1. **Scan your passport** via NFC using the IDenium mobile app (powered by ZKPassport)
2. **Generate a ZK proof** locally on your device — proves validity, age, nationality without revealing raw data
3. **Verify on-chain** on Starknet using Garaga — only a cryptographic nullifier is stored, no personal info
4. **Sign with IDenium** on any dApp — scan QR, authenticate with biometrics, done

## Quick Start

### Prerequisites

- Node.js 18+
- pnpm 9+
- Scarb 2.16+ (for Cairo contracts)

### Setup

```bash
# Clone the repo
git clone https://github.com/AnoukRImola/Idenium.git
cd Idenium

# Install dependencies
pnpm install

# Build all packages
pnpm build

# Start the web app
pnpm dev:web
```

Open [http://localhost:3000](http://localhost:3000) to see the landing page.

### Build Cairo Contracts

```bash
cd contracts
scarb build
```

### Deploy to Sepolia

```bash
# Configure your account first
sncast account create --name idenium --network sepolia

# Deploy
cd contracts
bash scripts/deploy.sh
```

## SDK Usage

Integrate "Sign with IDenium" in your dApp:

```typescript
import { IDenium, VerificationLevel } from "@idenium/sdk";

const idenium = new IDenium({
  network: "sepolia",
  registryAddress: "0x...",
  verifierAddress: "0x...",
});

// Check if a user is verified on-chain
const verified = await idenium.isVerified(userAddress);

// Create a verification request (generates QR code)
const request = await idenium.createVerificationRequest({
  requiredVerifications: [VerificationLevel.AGE_OVER_18],
  domain: "my-dapp.xyz",
});

// Wait for proof from mobile app
await idenium.waitForProof(request, (result) => {
  console.log("Verified!", result);
}, (error) => {
  console.error("Failed", error);
});
```

### React Component

```tsx
import { SignWithIDenium } from "@idenium/sdk/react";

<SignWithIDenium
  config={ideniumConfig}
  requestOptions={{
    requiredVerifications: [VerificationLevel.PASSPORT_VALID],
    domain: "my-dapp.xyz",
  }}
  onSuccess={(result) => console.log("Verified!", result)}
/>
```

## Smart Contracts

### IDeniumRegistry

Manages wallet ↔ nullifier mappings after ZK proof verification.

| Function | Description |
|----------|-------------|
| `register(proof, wallet)` | Verify proof on-chain and register wallet |
| `is_verified(address)` | Check if address is verified |
| `get_nullifier(address)` | Get nullifier for an address |
| `get_address_by_nullifier(nullifier)` | Reverse lookup |
| `get_total_verified()` | Total verified users |

### IDeniumVerifier

ZK proof verification contract. In production, this is generated by Garaga from Noir circuit verification keys. Currently runs in dev mode for hackathon demo.
